🗃️ Part 2 — SQLite

😱 The Problem: Your Data Disappears!

Your app saves data to a file called lms-data.json. But on Railway, every time you update and redeploy your app...

💥 The file gets wiped clean! All students, submissions, and scores — gone! This is because Railway's file storage is "ephemeral" (temporary).
🗃️ The fix? Use SQLite + a Volume!

SQLite is a tiny database that saves everything to one file. When you add a Volume on Railway, that file lives in a safe permanent place — it survives updates forever! ♾️

🗃️ SQLITE

🗃️ How It Works

💡 SQLite + Volume = Permanent Data

📂
Before (JSON)
Data in lms-data.json — disappears every redeploy 😢
🗃️
After (SQLite)
Data in lms.db on a Volume — lives forever 🎉

3 steps to set it up:

1Install the SQLite package on your computer
2Ask Claude to update your database.js file
3Add a Volume on Railway + set one env variable

SQLite Step 1 of 3

📦 Install the SQLite Package

Open Terminal, make sure you're in your project folder, then run this command:

Install better-sqlite3 (the SQLite tool for Node.js):
npm install better-sqlite3
# Wait for it to finish... takes about 30 seconds
📋 This adds better-sqlite3 to your package.json file automatically — so Railway knows to install it too when deploying!
Check it was installed correctly:
node -e "require('better-sqlite3'); console.log('SQLite OK! 🎉')"

SQLite Step 3 of 3

💾 Add a Volume on Railway

A Volume is like a permanent hard drive attached to your app. Data saved there survives every update.

1On railway.app, open your project
2Click "New" → click "Volume"
3Set the Mount Path to: /data
4Click "Create"
5Now go to Variables tab → add one more variable:
# Name → Value
DATABASE_PATH → /data/lms.db
